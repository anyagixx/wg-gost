# 📱 Настройка клиентов WireGuard

Это руководство поможет настроить WireGuard на различных устройствах.

---

## 📱 Мобильные устройства (iOS / Android)

### Шаг 1: Установка приложения

- **iOS (iPhone/iPad):** [WireGuard в App Store](https://apps.apple.com/app/wireguard/id1441195209)
- **Android:** [WireGuard в Google Play](https://play.google.com/store/apps/details?id=com.wireguard.android)

### Шаг 2: Получение конфигурации

1. Откройте веб-админку: `http://ВАШ_TIMEWEB_IP:51821`
2. Введите пароль
3. Нажмите **"New Client"**
4. Введите название устройства (например, `iphone-max`)
5. Нажмите **"Create"**

### Шаг 3: Импорт конфигурации

**Вариант А: QR-код (рекомендуется)**
1. В админке нажмите на QR-код рядом с клиентом
2. Откройте приложение WireGuard на телефоне
3. Нажмите "+" → "Scan QR-code"
4. Наведите камеру на QR-код
5. Дайте название подключению

**Вариант Б: Файл конфигурации**
1. Скачайте `.conf` файл в админке
2. Перешлите файл на телефон (email, мессенджер, облако)
3. Откройте приложение WireGuard
4. Нажмите "+" → "Create from file / Import"
5. Выберите файл

### Шаг 4: Подключение

1. В приложении WireGuard переключите тумблер на созданном туннеле
2. При первом подключении подтвердите запрос на создание VPN

## 💻 Ubuntu Desktop

### Шаг 1: Установка WireGuard

```bash
sudo apt update
sudo apt install wireguard resolvconf -y
```

### Шаг 2: Получение конфигурации

1. Откройте веб-админку: `http://ВАШ_TIMEWEB_IP:51821`
2. Создайте нового клиента (например, `ubuntu-laptop`)
3. Скачайте `.conf` файл в папку `Загрузки`

### Шаг 3: Импорт в Network Manager

```bash
# Импорт конфигурации
nmcli connection import type wireguard file ~/Загрузки/имя-клиента.conf
```

### Шаг 4: Отключение IPv6 (ВАЖНО!)

1. Откройте **Настройки** → **Сеть**
2. Найдите ваше VPN подключение и нажмите шестерёнку ⚙️
3. Перейдите на вкладку **IPv6**
4. Выберите метод: **Отключено (Disable)**
5. Нажмите **Применить**

### Шаг 5: Подключение

- Кликните на значок сети в правом верхнем углу
- Найдите ваше VPN подключение
- Нажмите **Подключиться**

## 🪟 Windows

### Шаг 1: Установка WireGuard

Скачайте с официального сайта: https://www.wireguard.com/install/

Или через командную строку (PowerShell от администратора):
```powershell
winget install WireGuard.WireGuard
```

### Шаг 2: Получение конфигурации

1. Откройте веб-админку: `http://ВАШ_TIMEWEB_IP:51821`
2. Создайте нового клиента (например, `windows-pc`)
3. Скачайте `.conf` файл

### Шаг 3: Импорт конфигурации

1. Запустите WireGuard
2. Нажмите **"Import tunnel(s) from file"**
3. Выберите скачанный `.conf` файл
4. Туннель появится в списке

### Шаг 4: Отключение IPv6 (ВАЖНО!)

1. Нажмите **Win + R**, введите `ncpa.cpl`
2. Найдите адаптер WireGuard (называется как туннель)
3. Правый клик → **Свойства**
4. **Снимите галочку** с "IP версии 6 (TCP/IPv6)"
5. Нажмите OK

### Шаг 5: Подключение

1. Откройте WireGuard
2. Выберите туннель
3. Нажмите **Activate**

## 🍎 macOS

### Шаг 1: Установка WireGuard

- [WireGuard в Mac App Store](https://apps.apple.com/app/wireguard/id1451685025)

Или через Homebrew:
```bash
brew install wireguard-tools
```

### Шаг 2: Получение и импорт конфигурации

Аналогично iOS — отсканируйте QR-код или импортируйте файл.

### Шаг 3: Отключение IPv6

1. **Системные настройки** → **Сеть**
2. Выберите VPN подключение WireGuard
3. **Дополнительно** → вкладка **TCP/IP**
4. Настройте IPv6: **Выключено** (если доступно)
5. Или добавьте в конфиг:
   ```
   [Interface]
   ...
   DNS = 1.1.1.1
   ```
   И удалите любые IPv6 адреса

## ✅ Проверка работоспособности

После подключения проверьте:

| Сервис | Ожидаемый результат |
|--------|---------------------|
| **2ip.ru** | Русский IP (Timeweb) — это нормально! |
| **YouTube** | Видео грузятся без buffering |
| **Telegram** | Медиафайлы отправляются мгновенно |
| **Instagram** | Лента и сторис грузятся быстро |
| **Банк/Госуслуги** | Работают без проблем |

### Командная строка для проверки (Linux/macOS)

```bash
# Проверка IP (должен быть русский)
curl -s ifconfig.me

# Проверка DNS (должен идти через 1.1.1.1)
nslookup google.com

# Проверка маршрута к YouTube
traceroute youtube.com
```

## ❓ Устранение неполадок

### YouTube не работает

1. **Отключите IPv6** на адаптере WireGuard
2. Переподключите VPN
3. Очистите кэш браузера (Ctrl+Shift+Delete)

### Медленные медиа в Telegram

1. Закройте Telegram полностью
2. Откройте заново (сброс подключений)
3. Проверьте, что IPv6 отключён

### Не открываются сайты

1. Проверьте статус туннеля в приложении
2. Попробуйте переподключиться
3. Проверьте интернет без VPN

### "Handshake did not complete"

1. Проверьте правильность IP-адреса сервера
2. Убедитесь, что порт 51820 UDP не заблокирован
3. Перезапустите WireGuard на сервере: `docker restart wg-easy`

## ⚡ Оптимизация производительности

### 🔧 MTU 1280 — настроен автоматически!

WireGuard Easy автоматически добавляет `MTU = 1280` в каждый генерируемый конфиг. Это оптимизировано для работы через GOST-туннель и обеспечивает:
- Быструю отправку файлов в Telegram Desktop
- Отсутствие фрагментации пакетов
- Стабильную работу всех приложений

**Ничего настраивать вручную не нужно!** Просто скачайте конфиг из админки и импортируйте его.

### 🔗 Совместимость с OpenVPN (сеть 10.0.0.0/8)

**Проблема:** При одновременном подключении к OpenVPN и WireGuard недоступны серверы во внутренней сети (например, `10.0.12.2`).

**Причина:** WireGuard с `AllowedIPs = 0.0.0.0/0` перехватывает ВЕСЬ трафик, включая `10.x.x.x`.

**Решение:** Исключить подсеть `10.0.0.0/8` из WireGuard.

#### Изменение AllowedIPs

Откройте конфиг WireGuard в текстовом редакторе и замените:
```
AllowedIPs = 0.0.0.0/0
```

На:
```
AllowedIPs = 0.0.0.0/5, 8.0.0.0/7, 11.0.0.0/8, 12.0.0.0/6, 16.0.0.0/4, 32.0.0.0/3, 64.0.0.0/2, 128.0.0.0/1
```

Это математически охватывает весь интернет IPv4, **КРОМЕ** `10.0.0.0/8`.

#### Linux (NetworkManager)

1. Настройки → Сеть → шестерёнка у VPN
2. Вкладка IPv4 → Маршруты
3. Удалите маршрут `0.0.0.0/0` и добавьте подсети из списка выше
4. Или отредактируйте файл в `/etc/NetworkManager/system-connections/`

#### Windows

1. WireGuard → правый клик на туннель → **Edit**
2. Замените `AllowedIPs`
3. Сохраните и переподключите

## 🔧 Продвинутые настройки

### Split-tunneling на клиентах

Если вы хотите направить в VPN только определённые сайты, отредактируйте `AllowedIPs` в конфиге:

**Только Google/YouTube:**
```
AllowedIPs = 142.250.0.0/15, 172.217.0.0/16, 173.194.0.0/16
```

**Весь трафик (по умолчанию):**
```
AllowedIPs = 0.0.0.0/0
```

### Изменение DNS

В конфиге клиента найдите строку `DNS` и измените:
```
DNS = 8.8.8.8  # Google DNS
DNS = 1.1.1.1  # Cloudflare DNS (по умолчанию)
DNS = 77.88.8.8  # Яндекс DNS
```
